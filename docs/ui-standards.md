# AxData Web UI 规范

这页约束 AxData Web 的产品表达。核心目标是让用户一眼分清：接口是临时查一次，采集是把数据保存到本机，数据页是查看已经落盘的数据。

## 总原则

- 不做营销页，开始页是开发文档入口。
- 页面文案直接、清晰，少用实现细节词。
- 接口、采集、数据、插件四条线必须分开。
- 不用假样例，不在页面打开时为了展示样例实时请求源端。
- UI 控件要贴合输入类型：日期用日期框，枚举用选择器，多格式用多选，布尔项用开关。
- 不把核心操作藏在大片说明文字里。

## 接口页

接口页回答：“这个源端接口怎么临时调用，返回什么？”

必须展示：

- 接口名称、中文名和数据源。
- 插件 / Provider 状态。
- 输入参数。
- 输出字段。
- 真实静态样例。
- 默认 curl 调用示例。
- 本地 SDK 与远程 SDK/API 的边界。

内容来源：

- request 接口页由 Registry runtime catalog 生成，Web 只做通用渲染。
- 左侧目录来自插件 / Provider 的 `menu_path`。
- 中文摘要、正文说明、参数提示和参数示例来自 `summary_zh`、`description_zh`、`params_note_zh`、`params_example_zh`。
- 参数和字段来自 `parameters`、`fields`。
- 真实返回样例来自静态 `example.response`。
- 类别码、枚举映射、字段口径说明等参考表来自 `reference_sections`。
- Web 不为具体 request 接口维护静态页面副本；保留特殊非 request 页面时要能说明原因。

样例规则：

- 样例来自真实请求过的静态快照。
- 页面打开不实时请求源端。
- 返回字段必须和字段声明对齐。
- 如果样例为空，必须写清楚原因。

远程 token 说明只放在远程 SDK/API 附近，必须写清“不影响本地 SDK 调用”。本地 `AxDataClient()` 不需要 AxData HTTP token。

## 采集页

采集页回答：“如何把数据保存到本机，并管理任务？”

推荐结构：

1. 基本参数
2. 输入参数
3. 输出参数
4. 创建任务
5. 任务列表
6. 运行进度
7. 最近记录
8. 日志/诊断

基本参数包含：

- 任务名称。
- 输出格式，多选，Parquet 默认选中且不可取消。
- 存储路径，只显示目录。
- 文件名规则，如果要暴露文件名，就单独一行。
- 并发档位。
- 自定义服务器数和每服务器长连接数。

输入参数默认展开，用户应该能直接在输入框、选择器或日期控件里配置。输出参数可以默认展开，尤其当页面视觉需要上下平衡时。

内容来源：

- 采集器列表和详情来自 CollectorRegistry 返回的 CollectorSpec。
- 输入参数优先来自 CollectorSpec 的 `default_params` / 参数声明；如果采集器声明了关联接口，可以复用 runtime interface catalog 的参数说明。
- 输出字段、写入策略和质量规则来自 CollectorSpec 的 `output` / `quality` / 数据集声明；如果采集器声明了关联接口，可以复用 runtime interface catalog 的字段说明。
- Web 不为具体采集器维护静态页面副本，也不回查旧接口静态 catalog 作为采集器详情来源。

创建任务按钮放在输入/输出配置之后。任务创建后进入任务列表，手动采集和定时采集都在任务列表里操作。

任务列表应是紧凑一行式，不要堆很多说明卡片。每行至少包含：

- 任务名。
- 输出路径摘要。
- 手动采集按钮。
- 定时类型。
- 定时时间。
- 定时开关。
- 最近状态。
- 删除任务按钮。

文案使用“手动采集”，不要用容易误解的“采集一次”。如果选择定时采集，应先设置定时时间。A 股行情类任务默认使用“交易日每天”。

最近记录默认展示最近 3 条。每条记录应支持单独删除，删除前确认。删除运行记录不删除任务配置，也不删除已采集数据。

## 数据页

数据页回答：“本机已经有什么数据，可以怎么查？”

数据页不请求源端，不创建采集任务。必须展示：

- 数据集中文名和 `dataset_id`。
- 来源采集器或来源说明。
- 层级。
- 行数。
- 日期范围。
- 最近更新时间。
- 质量状态。
- 支持格式。
- 数据路径和预览来源。

数据页只能列已经有本地输出的数据集。插件 manifest、CollectorSpec、`queryable=true` 或预期 `storage_path` 只是声明；如果没有真实 output path、metadata、run/downloader output 记录或 Parquet 文件，就不应出现在数据页列表，也不应提供删除 dataset 操作。

预览默认读已入库 Parquet。DuckDB 可以作为查询执行层或查询缓存，但页面不要把它描述成主存储或 Parquet 备份。

删除数据集前必须弹窗，告诉用户将删除哪个数据根目录下的哪一段数据。

## 插件页

插件页回答：“当前安装了哪些能力，它们提供什么接口和采集器？”

必须区分：

- Provider 数据源能力。
- Collector 采集能力。
- DownloaderProfile 兼容能力。
- 预装、AXP、pip、editable 等安装来源。

插件卡片可以提供“导出 AXP”。该动作只导出插件安装包，不导出已采集数据、metadata、任务历史、日志、缓存或 token；导出后仍由 manifest 判断数据源/采集器能力。

诊断项必须有中文解释，不能只显示英文 key。删除插件时需要说明：

- 会隐藏或移除哪些接口。
- 会隐藏或移除哪些采集器。
- 不会删除哪些本地数据。
- 预装插件是逻辑卸载还是物理删除。

## 开始页

开始页是开发文档入口，短入口在上面，长文档在下面。建议栏目：

- 快速开始。
- 数据源开发。
- 采集器开发。
- 数据集声明与 DuckDB。
- UI 规范。
- 打包安装。
- 插件排错。
- 架构说明。
- 服务器使用。
- Python SDK 主入口。

首页可以有架构图，但要服务理解，不要变成装饰图。
